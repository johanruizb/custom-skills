# HTML Report Specification

The optional HTML report is a self-contained, navigable document with filtering capabilities. Generated by `scripts/generate-html-report.py`.

## Requirements

- **Self-contained**: all CSS/JS inline or via CDN. No local file dependencies.
- **No external services**: works offline once loaded (CDN is acceptable for initial load but report must render without server calls).
- **Navigable**: table of contents, internal anchor links to each finding.
- **Filterable**: client-side filters by category, severity, confidence, module, status.
- **Evidence display**: code snippets with syntax highlighting for each finding.
- **Print-friendly**: reasonable print layout.

## Styling

- Use Tailwind CSS via CDN (`<script src="https://cdn.tailwindcss.com"></script>`) for utility classes and rapid layout.
- Syntax highlighting: use Prism.js or highlight.js via CDN with a dark theme for code evidence blocks.
- Badges: colored pills for severity (red=critical, orange=high, yellow=medium, blue=low, gray=info) and confidence (green=confirmed, yellow=probable, gray=hypothesis).
- Responsive: works on mobile, tablet, and desktop.

## Filter Implementation

Vanilla JS, no framework needed:

```javascript
// Filter state
const filters = {
  category: 'all',
  severity: 'all',
  confidence: 'all',
  module: 'all',
  status: 'all',
  search: ''
};

// Apply filters to finding cards
function applyFilters() {
  document.querySelectorAll('[data-finding]').forEach(el => {
    const match = (
      (filters.category === 'all' || el.dataset.category === filters.category) &&
      (filters.severity === 'all' || el.dataset.severity === filters.severity) &&
      (filters.confidence === 'all' || el.dataset.confidence === filters.confidence) &&
      (filters.module === 'all' || el.dataset.module === filters.module) &&
      (filters.status === 'all' || el.dataset.status === filters.status) &&
      (filters.search === '' || el.textContent.toLowerCase().includes(filters.search.toLowerCase()))
    );
    el.style.display = match ? 'block' : 'none';
  });
}
```

## Generation

The report is generated by `scripts/generate-html-report.py`, which takes a JSON state file as input and produces a self-contained HTML file. See the script for implementation.

Usage:
```bash
python scripts/generate-html-report.py --state .audit-state.json --output /tmp/audit-report-<timestamp>.html
```

The script reads the state file (see `templates/audit-state.json`), extracts findings and metadata, and renders the full HTML report.
